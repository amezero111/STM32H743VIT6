 遥控器接收模块 (Remote) 重构说明
为了在 FreeRTOS 系统下最大限度利用 CPU，遥控器数据解析已完全摒弃阻塞式与单字节中断读取，采用了极低耦合的 **“非阻塞硬件 DMA + RTOS 任务通知”** 架构。
**核心特性：
* **零系统开销**：使用 `HAL_UARTEx_ReceiveToIdle_DMA`，由 USART6 DMA 硬件后台静默搬运定长 18 字节的数据帧，期间不占用任何 CPU，单帧传输结束后才触发一次空闲中断。
* **RTOS 深度解耦**：空闲中断触发后仅向任务队列发送唤醒信号 (`osThreadFlagsSet`) 即刻重启 DMA；实际的大端转小端数据解算 (`Remote_Parse`) 会交由独立的 FreeRTOS 任务完成，中断极度轻量，绝不影响实时控制。
* **极简式调用**：
  1. 应用初始化层调用 `RemoteControlInit();` 即可自动绑定 UART6 与中断任务。
  2. FreeRTOS 解算任务(`StartRemote`)的死循环内调用 `RemoteControlTask();` 将自动实现挂起与阻塞解析，等待帧到来。
* **实时调试窗口**：应用层可直接将 `test_remote` 或 `remote_dev` 指针加入 Debug 模式的 Watch 变量窗口，随时展开点选并观测 `data` 内摇杆、拨轮、开关的实时整型数据变化。
